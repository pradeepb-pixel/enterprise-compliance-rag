## 🧠 Overview

A **production-grade Retrieval-Augmented Generation (RAG) platform** built with **Claude API** and **AWS Bedrock Knowledge Bases** — enabling enterprise teams to search, query, and extract insights from large compliance document repositories using natural language.

Deployed to handle regulatory document search and compliance query resolution across thousands of legal and policy documents.

## 🚀 Getting Started

### Prerequisites

```bash
Python 3.10+
AWS Account with Bedrock + S3 access
AWS CLI configured
Anthropic Claude API access via Bedrock
```

### Installation

```bash
# Clone the repo
git clone https://github.com/pradeepb-pixel/enterprise-compliance-rag.git
cd enterprise-compliance-rag

# Create virtual environment
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt
```

### Configuration

```bash
# Set your AWS credentials
export AWS_ACCESS_KEY_ID=your_key
export AWS_SECRET_ACCESS_KEY=your_secret
export AWS_DEFAULT_REGION=us-east-1

# Set Bedrock Knowledge Base ID
export KNOWLEDGE_BASE_ID=your_kb_id
```

### Run

```bash
# Ingest documents into knowledge base
python ingest.py --docs-path ./documents/

# Start the query interface
python main.py
```

## 🔄 How It Works

**1. Document Ingestion** — Compliance docs uploaded to S3 → chunked → embedded → stored in Bedrock Knowledge Base

**2. User Query** — Compliance team member asks a natural language question

**3. Semantic Retrieval** — Bedrock Knowledge Base finds the most relevant document chunks via vector search

**4. Context Injection** — Retrieved chunks injected into Claude API prompt

**5. Answer Generation** — Claude generates an accurate, cited answer grounded in the documents

**6. Source Returned** — User receives the answer + exact source document references
